# 🌳 WHY TREE — Execution Failure (Aligned with PRD)

## 1. Problem Statement

사용자는 해야 할 일을 떠올리지만 실행하지 않는다.

문제는 생산성이나 계획이 아니라:

👉 “실행까지 이어지지 않는 구조”다

---

## 2. WHY Breakdown

### Level 1
왜 실행하지 않는가?

---

### Level 2

1. 입력하지 않는다  
2. 쪼개지 못한다  
3. 실행하지 않는다  

---

## 3. Root Causes

### 3.1 입력하지 않는다

왜 입력하지 않는가?

- 앱을 열기 귀찮다  
- 정리해서 쓰기 어렵다  
- 생각이 금방 사라진다  

👉 Insight:
입력이 느리면 기록 자체를 안 한다

---

### 3.2 쪼개지 못한다

왜 쪼개지 못하는가?

- goal을 task로 변환하기 어렵다  
- 어디서부터 시작할지 모른다  

👉 Insight:
“실행 가능한 단위”가 아니면 행동이 시작되지 않는다

---

### 3.3 실행하지 않는다

왜 실행하지 않는가?

- 시작하기 귀찮다  
- 강제력이 없다  
- 실패해도 손해가 없다  

👉 Insight:
행동은 의지가 아니라 “강제 구조”로 만들어진다

---

## 4. Root Cause Summary

문제는 의지가 아니다.

실제 원인은:

1. 입력 마찰
2. 실행 단위 부재
3. 강제력 부재

---

## 5. Product Mapping (PRD 연결)

- 입력 마찰  
  → Quick Capture (텍스트 / 음성 입력)

- 실행 단위 부재  
  → AI Task Parser + Decomposition

- 강제력 부재  
  → Action Trigger + Penalty System

---

## 6. Conclusion

이 제품은 생산성 앱이 아니다.

👉 실행을 강제하는 시스템이다
